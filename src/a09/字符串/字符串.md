# 🧩 深入解析：Java 字符串拼接的内存玄机

> **核心知识点**：String 的不可变性、字符串常量池 (String Pool)、编译期优化。
> **测试代码**：
> ```java
> String s1 = "a";
> String s2 = s1 + "b";  // 运行时拼接
> String s3 = "a" + "b"; // 编译期拼接
> 
> System.out.println(s2 == "ab"); // false
> System.out.println(s3 == "ab"); // true 
> ```

在 Java 中，使用 `==` 判断两个对象时，**比较的永远是它们在内存中的地址（门牌号），而不是内容**。

下面我们像 JVM 一样，一步步推演这 5 行代码在内存中到底发生了什么。

---

## 🎬 内存推演全过程

### 第一步：`String s1 = "a";`

*   **场景描述**：使用双引号字面量直接创建字符串。
*   **内存动作**：
    1.  JVM 检查**字符串常量池 (String Pool)**，发现没有内容为 `"a"` 的对象。
    2.  在常量池中开辟空间，创建对象 `"a"`。
    3.  在栈内存中声明变量 `s1`，并将其指向常量池中 `"a"` 的**内存地址**。

### 第二步：`String s2 = s1 + "b";` 为什么是 false？

*   **场景描述**：一个**变量 (`s1`)** 与一个**字面量常量 (`"b"`)** 进行拼接。
*   **内存动作 (运行时行为)**：
    1.  因为涉及变量，拼接动作发生在**程序运行期间 (Runtime)**。
    2.  Java 底层为了处理变量拼接，会自动帮你偷偷创建一个 `StringBuilder` 对象。
    3.  实际底层执行的逻辑是：`new StringBuilder().append(s1).append("b").toString();`
    4.  **关键点在于 `toString()`**：`StringBuilder` 的 `toString()` 方法底层实现是 `new String("ab")`。
    5.  只要使用了 `new` 关键字，就意味着会在**堆内存的普通区域**（而非字符串常量池）中开辟一块全新的空间，造出一个内容为 `"ab"` 的新对象。
    6.  最后，变量 `s2` 拿到的，是这个**堆内存普通区域中新对象的地址**。

### 第三步：`String s3 = "a" + "b";` 为什么是 true？

*   **场景描述**：两个**纯字面量常量**（写死的字符串）进行拼接。
*   **内存动作 (编译期优化)**：
    1.  Java 编译器（`javac`）有一个极其聪明的特性叫做 **常量折叠 (Constant Folding)**。
    2.  在把 `.java` 编译成 `.class` 字节码文件时，编译器发现 `"a"` 和 `"b"` 都是固定不变的死值。
    3.  为了提升运行效率，编译器直接在**编译阶段**就把它们合并成了 `"ab"`。
    4.  因此，在实际运行时，这行代码等同于：`String s3 = "ab";`
    5.  此时，JVM 拿着 `"ab"` 去**常量池**里找。如果发现常量池里还没有，就在常量池中创建一个 `"ab"`；如果发现已经有了，就直接指向它。
    6.  变量 `s3` 拿到的，是**常量池中那个独一无二的 `"ab"` 的地址**。

### 第四步：对比结果剖析

#### 1. 解释 `System.out.println(s2 == "ab"); // false`
*   `"ab"`（右侧的字面量）：JVM 会去**常量池**中获取或创建，它代表的是常量池中的地址。
*   `s2`：根据第二步推演，它指向的是**堆内存普通区域**中 `new` 出来的新对象地址。
*   **结论**：一个是堆内存的地址，一个是常量池的地址。**门牌号不同，用 `==` 比较自然是 `false`。**

#### 2. 解释 `System.out.println(s3 == "ab"); // true`
*   `"ab"`（右侧的字面量）：代表**常量池**中的地址。
*   `s3`：根据第三步推演，经过编译期优化后，它本身就是通过双引号直接声明的 `"ab"`，指向的也是**常量池**中的那个同一个地址。
*   **结论**：双方指向的都是常量池里同一个对象的内存地址。**门牌号完全一致，用 `==` 比较自然是 `true`。**

---

## 💡 开发铁律 (Best Practice)

通过这个底层推演，我们得出一个 Java 开发中最基本也是最重要的原则：

**判断两个字符串的【内容】是否相等，永远、绝对、必须使用 `equals()` 方法！**
**千万不要使用 `==`！** （除非你明确知道你就是在比较它们是不是同一个内存地址）。

*   例如：`s2.equals("ab")` 的结果绝对是 `true`，因为它会逐个字符去比对内容，而无视内存地址的差异。
